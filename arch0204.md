# Cerberus Architecture (02-04-2026)

## Overview
Cerberus is a GCP Development Environment Guardian designed to correlation three signals: **Resource Ownership**, **Resource Cost**, and **Personnel Status**. The system uses a LangGraph-based agentic loop to automate the discovery, classification, and cleanup of idle or orphaned resources while maintaining a strict human-approval gate.

## 1. Core Architecture: Agent Loop
The system is built on a **LangGraph ReAct-style agent**. Unlike a static pipeline, the agent can adapt its reasoning based on the signals it discovers (e.g., if labels are missing, it can search IAM history or audit logs).

### Graph Nodes
- **`scan_node`**: Discovers resources (VMs, Disks, SQL instances, IPs) in the target GCP project.
- **`enrich_node`**: Correlates resources with IAM ownership, last activity timestamps, and current billing estimates.
- **`reason_node`**: Uses Gemini 1.5 Pro to classify each resource:
  - `safe_to_stop`: Resource is idle but owned.
  - `safe_to_delete`: Resource is orphaned or abandoned.
  - `needs_review`: Ownership or usage signals are ambiguous.
- **`approve_node`**: A blocking interrupt that waits for a human to select which actions to execute.
- **`revalidate_node`**: Re-checks the GCP state *immediately* before execution to prevent acting on stale data (drift detection).
- **`execute_node`**: Performs the approved mutations (stop, delete, or archive to Coldline storage).
- **`audit_node`**: Records the final outcomes to a local JSON log and flushes the LangSmith trace.

## 2. Multi-Head Specialized Logic
Beside the main cleanup graph, Cerberus implements specialized "Heads":
- **IAM Head**: Decomposes natural language access requests into minimum-privilege GCP permissions and manages the lifecycle of IAM tickets.
- **Cost Head**: Periodically aggregates potential savings and actual recovered savings.
- **Security Head**: (Candidate C) Planned for advanced data classification and sensitive disk archival.

## 3. Data Integrity & Safety
- **LangSmith Tracing**: Every reasoning step and tool call is captured for auditability.
- **Human-in-the-Loop (HITL)**: No mutation occurs without explicit user approval.
- **State Serialization**: LangGraph state is preserved at the approval gate, allowing for asynchronous human intervention.
- **Dry-Run Mode**: The system defaults to `dry_run=True` to prevent accidental infrastructure changes during development and demos.

## 4. Technology Stack
- **Backend**: Python 3.10+, FastAPI.
- **Orchestration**: LangGraph, LangSmith.
- **LLM**: Gemini 1.5 Pro (Vertex AI).
- **State/Persistence**: In-memory LangGraph Checkpointer + ChromaDB for long-term IAM history and audit logs.
- **Frontend**: React, Vite, TypeScript, TailwindCSS.
